### What I work on

I'm a final-year CS student who likes hard problems with tangible output — things you can
*demo*, not just describe. Most of my work lives in TypeScript and Python, and lately I've
been drawn to applying ML and computer vision to problems in education, accessibility, and
healthcare.

I care about shipping things that work, writing them down clearly, and improving them based
on how people actually use them.

### Selected projects

**🎸 [strings-guitar](https://github.com/francisreubenr-rvu/strings-guitar)** — `TypeScript`
A webcam-based guitar learning tool. Uses real-time hand-landmark tracking to detect finger
placement, recognize chords, and guide learners through a structured curriculum. Built to
remove the "am I doing this right?" friction that stops most beginners.

**🤟 [SignBridge](https://github.com/francisreubenr-rvu/SignBridge)** — `Python`
A sign-language recognition system aimed at closing communication gaps between signing and
non-signing users. An exercise in real-time gesture classification and accessibility-first design.

**🏥 [Emet](https://github.com/francisreubenr-rvu/Emet)** — `Python`
A healthcare-focused application built around making clinical/personal health workflows simpler.

**📊 [scholar-dashboard](https://github.com/francisreubenr-rvu/scholar-dashboard)** — `TypeScript`
An academic productivity dashboard for tracking coursework, deadlines, and progress.

**🎓 [kcet-cutoff-explorer](https://github.com/francisreubenr-rvu/kcet-cutoff-explorer)** — `HTML`
A tool to explore KCET admission cutoffs — turning messy public data into something students
can actually navigate when choosing colleges.

### A note on how I build

2. **Ship a rough version early**, then iterate from real feedback.
3. **Write it down** — a project isn't done until someone else can understand it.
